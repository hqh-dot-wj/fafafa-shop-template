# Doc Request Flow

## 目标

处理“是否创建或修改文档”这类请求时，确保遵守仓库的文档确认规则。

## 核心规则

- 新增或生成 `.md` 文档前，必须先征得用户确认。
- 未确认前，只能给：
  - 提纲
  - 蓝图
  - 改造清单
  - 章节结构建议

## 已确认后的动作

确认后再判断文档类型：

- 正式需求文档
- 正式设计文档
- 轻量实施说明
- 规范重构文档

## 默认策略

- 能用轻量文档解决，就不要默认走重型模板
- 只有用户明确要求 PRD / 设计文档 / 分析报告时，再使用较完整模板

## 写文档前自查

- 该文档是否真的需要落盘
- 应该放在哪一层目录
- 是否会和现有文档重复
- 是否引用权威源而不是复制内容
