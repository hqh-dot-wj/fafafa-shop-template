# Backend Requirements Navigation

Module-level requirement drafts under this directory were removed after their useful acceptance rules were covered by code, OpenAPI/generated types, frontend-backend alignment docs, or `apps/backend/docs/process-specs/**`.

## Maintenance Rules

1. Do not recreate broad requirement handbooks for implemented behavior.
2. Put enduring acceptance rules in `../process-specs/`, interface truth in OpenAPI/generated types, and domain vocabulary in `docs/domain/`.
3. Create a new requirement doc only when a business decision is pending; delete or migrate it after delivery closes.
