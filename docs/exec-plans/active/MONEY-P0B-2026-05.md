# Exec Plan: MONEY-P0B-2026-05

实施 `MONEY_PRECISION_PROTOCOL` 的 P0-B：后端金额精度底座。P0-A 与 P1-A 已在工作区实施，本 plan 不重做、不回滚。

## 1. 目标 / 非目标

**目标**

- 新增 `@libs/common-utils/money`，覆盖 `add/sub/mul/div/sum/toFen/format` 与守恒拆分。
- 微信支付、退款、分账、商家转账金额入参统一整数分校验，禁止裸 number 和小数分。
- 修复 P0-B 覆盖的后端金额点位：优惠计算、checkout payAmount、支付回调校验、积分换算、佣金部分退款守恒、提现手续费、对账差异。
- 补 Money 单测、`toFen` 边界测试、关键业务金额回归测试。

**非目标**

- 不做 P1-B/P1-C/P1-D/P1-E。
- 不改已有 migration，不改 schema/seed，不做历史数据回填。
- 不改前端金额工具，不改 OpenAPI 契约，不执行 `generate-types` 作为完成条件。
- 不改变业务口径，只改变计算精度和边界校验。
