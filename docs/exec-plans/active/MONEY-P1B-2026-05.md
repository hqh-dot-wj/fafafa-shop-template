# Exec Plan: MONEY-P1B-2026-05

实施 `MONEY_PRECISION_PROTOCOL` P1-B：平台 `REFUND` 对账与渠道金额拆口径。

## 1. 目标 / 非目标

**目标**

- 新增平台对账 `REFUND` scope。
- `REFUND` 本地事实源只读取 `FinRefund`，不从 `OmsOrder.status` 反推。
- 渠道账单明细拆分用户退款、结算退款、退款手续费、优惠退款、净额。
- 对账结果能解释 `19.90` 用户退款、`0.02` 手续费、`19.88` 净额。
- 服务层 / 对账逻辑测试覆盖 `MATCHED` / `UNMATCHED` / `BUFFERED` / `IGNORED`。

**非目标**

- 不做 P1-C 结算调整单。
- 不做 P1-D 门店财务净额化。
- 不做 P1-E 前端金额工具。
- 不重做 P0-A / P1-A。
- 不修改已有 migration，不做历史真实回填。
